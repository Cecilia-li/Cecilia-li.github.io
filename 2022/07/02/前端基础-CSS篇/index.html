Flex 布局

阮一峰的flex系列

1、基本概念

采用Flex布局的元素，称为Flex容器（flex container），简称”容器”。它的所有子元素自动成为容器成员，称为Flex项目（flex item），简称”项目”。

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。主轴的开始位置（与边框的交叉点）叫做main start，结束位置叫做main end；交叉轴的开始位置叫做cross start，结束位置叫做cross end。

项目默认沿主轴排列。单个项目占据的主轴空间叫做main size，占据的交叉轴空间叫做cross size。

2、容器的属性

属性名 用途 取值
flex-direction 决定主轴方向 row（默认）、row-reverse、column、column-reverse
flex-wrap 定义如果一条轴线排不下，如何换行 no-wrap（默认）、wrap、wrap-reverse
flex-flow flex-direction和flex-wrap的简写形式 默认值为row no-erap
justify-content 主轴上的对齐方向 flex-start（默认）、flex-end、center、space-between、space-around
align-items 交叉轴上的对齐方向 flex-start、flex-end、center、baseline、stretch（默认）
align-content 定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用。 flex-start（默认）、flex-end、center、space-between、space-around、 、 stretch `（默认）

BFC

1、什么是BFC？

Block Format Context（块级格式上下文）。

2、触发条件：

  1. 根元素；
  2. float值不为none；
  3. position为absolute或fixed；
  4. overflow不为visible；
  5. display为flex/inline-block/table-cell/table-caption。

3、应用：

  • BFC中浮动元素高度也会参与计算，可用于防止浮动导致父元素高度塌陷（清除浮动）；
  • 避免上下边距重叠。

box-sizing属性：

  • content-box：默认值。此时为标准盒模型；content = 内容宽度
  • border-box：IE盒模型；content = 内容宽度 + padding + border

CSS选择器

1、CSS选择器有哪些？

  1. 简单选择器，如id选择器、类选择器、标签选择器；
  2. 组合器选择器，如后代选择器li a、子选择器>、相邻兄弟选择器+、通用兄弟选择器~；
  3. 伪类选择器，如a:hover、a:link、a:visited、a:active、p:nth-child(n)；
  4. 伪元素选择器，如p::after、p::before；
  5. 属性选择器，如a[attribute="value"]。

1.1 后代选择器(div p)

匹配属于指定元素后代的所有元素。包括但不仅限于子元素。

1.2 子选择器(div > p)

匹配属于指定元素子元素的所有元素（仅一层元素）。

1.3 相邻兄弟选择器(div + p)

匹配所有作为指定元素的相邻同级的元素。

1.4 通用兄弟选择器(div ~ p)

匹配属于指定元素的同级元素的所有元素。

1.5 伪类与伪元素的区别

伪类的效果可以通过添加一个实际的类来达到，
而伪元素的效果则需要通过添加一个实际的元素才能达到。

2、优先级

!important > [ id > class > tag ]，同时遵循就近原则，!important比内联优先级高。

3、属性的继承

可继承的属性：font-size, font-family, color；

不可继承的样式：border, padding, margin, width, height。

4、at规则

css除了普通的选择器 + 声明块的规则外，还有一种@规则。用来描述样式使用的字符集、引入外部样式、条件判断等等。

  • @charset：定义样式表使用的字符集。
  • @import：告诉 CSS 引擎引入一个外部样式表。
  • @media：媒体查询。
  • @keyframes：用于定义animation的动画帧。

position属性

  • static静态定位，position的默认属性，按照正常文档流进行排列；
  • relative相对定位，参考自身静态位置通过top，bottom，left，right定位；
  • absolute绝对定位，参考距其最近一个不为static的父级元素通过top，bottom，left，right定位；
  • fixed固定定位，以浏览器的可视窗口为参照点；
  • sticky粘性定位，可以被认为是相对定位和固定定位的混合。

Sass与Less

1、什么是CSS预处理器？

CSS预处理器是一种脚本语言，用一种专门的编程语言来进行Web 页面样式设计，然后再转换为正常的CSS样式，进而实现构建动态CSS样式。

CSS预处理器为CSS增加了一些编程的特性，无需考虑浏览器的兼容性问题。

Sass和Less就是两种常见的CSS预处理器，我们可以把Sass和Less编辑的样式转换为正常css样式。

2、Sass与Less的本质区别

  1. less是基于javascript运行，所以less是在客户端处理。sass是基于Ruby的，是在服务器端处理。
  2. 编写变量的方式不同。less使用@,而sass使用$。
  3. 在less中，仅允许循环数值。在Sass中，我们可以遍历任何类型的数据；但在Less中，我们只能使用递归函数循环数值。
  4. Sass有Compass，Less有Preboot。都是用于集成mixins的扩展（在整个站点中存储和共享CSS声明的能力），LESS的软件支持比Sass更加分散，导致许多不同的扩展选项可能不会以相同的方式运行。对于项目，我们可能需要所有列出的扩展以获得与Compass类似的性能。

响应式布局

1、px、em、rem、%、vw、vh单位之间的区别

  • px就是pixel（像素）的缩写，相对长度单位，相对于屏幕分辨率。
  • rem是相对于根元素的相对长度单位，常用于响应式布局。
  • em是相对于父元素的相对长度单位。
  • %相对于** 包含块 **（有父元素相对于父元素，无父元素相对于视口，或者继承于父元素）的计量单位。
  • vw和vh由视窗（Viewport）大小来决定的，单位1，代表类似于1%。

2、响应式布局常用方案

  • 媒体查询，根据不同屏幕宽度设置根元素font-size；之后设置rem，基于根元素计算相对单位。
  • vw/vh

3、rem弊端

rem弊端：如果使用媒体查询设置，结果呈现阶梯状，在某一阶段，无法改变。

4、网页视口尺寸

  • window.screen.height // 屏幕高度
  • window.innerHeight //网页视口高度
  • document.body.clientHeight //body高度

vh：网页视口高度window.innerHeight的1/100；

vw：网页视口宽度window.innerWidth的1/100；

vmax：取vh和vw中的最大值；vmin：取vh和vw中的最小值。

margin

margin取负值？

  • margin-top、margin-left负值，元素分别上移、左移；
  • margin-bottom负值，下方元素上移，自身不受影响；
  • margin-right负值，右侧元素左移，自身不受影响。

line-height如何继承？

例：

line-height值从父标签中已设置该值的元素继承，继承规则：

  • line-height为具体数值，如30px，直接继承该值；
  • 写比例，如2/1.5，则继承该比例（14px*2）;
  • 写百分比，如200%，则继承计算出来的值（20px*200%)。
