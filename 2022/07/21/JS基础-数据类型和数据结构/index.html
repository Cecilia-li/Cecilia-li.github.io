JS 基础 - 数据类型和数据结构

Author: Cecilia Li Date: July 21, 2022  19:25:31 Category: 知识体系整理

一、js中的八种数据类型

js中的数据类型分为基本数据类型和引用数据类型

  • 基本数据类型：Number、String、Boolean、BigInt、Symbol、Null、Undefined；
  • 引用类型（即对象类型Object）：如Object、Function、Array、Date。

二、js语言的底层数据结构

JavaScript 中的变量在内存中的具体存储形式

  • JavaScript基本类型数据都是直接按值存储在栈中的，每种类型的数据占用的内存空间的大小是确定的，并由系统自动分配和自动释放。这样带来的好处就是，内存可以及时得到回收，相对于堆来说 ，更加容易管理内存空间。
  • JavaScript引用类型数据的地址指针是存储于栈中的，当我们想要访问引用类型的值的时候，需要先从栈中获得对象的地址指针，然后，在通过地址指针找到堆中的所需要的数据。

三、Null 和 Undefined

1、Null

MDN 上介绍：值null特指对象的值未设置。null是JavaScript基本类型之一，在布尔运算中被认为falsy。

null指示变量未指向任何对象。

2、Undefined

MDN 称：全局属性undefined表示原始值undefined。它是JavaScript的原始数据类型。

对于一个已经定义了，但未赋值但变量，系统默认其类型为Undefined类型，值也为undefined。

一个函数如果没有使用return指定返回值，就会返回一个undefined值。

五、Set

1、简介

Set对象是值的集合，你可以按照插入的顺序迭代它的元素。Set 中的元素只会出现一次，即Set中的元素是唯一的。

1.1 值的相等

Set内部判断两个值是否不同，使用的算法叫做“Same-value-zero ”，它类似于精确相等运算符（===），主要的区别是向Set 加入值时认为NaN等于自身，而精确相等运算符认为NaN不等于自身。

2、constructor

==Set()==：创建一个新的Set对象。

  • Set.prototype.size：返回Set实例的成员总数。

4、实例方法

Set实例的方法分为两大类：操作方法（用于操作数据）和遍历方法（用于遍历成员）。下面先介绍四个操作方法。

操作方法

  • Set.prototype.add(value)：在Set对象尾部添加一个元素。返回该Set 对象。
  • Set.prototype.delete(value)：移除值为value 的元素，并返回一个布尔值来表示是否移除成功。
  • Set.prototype.has(value)：返回一个布尔值，表示该值在Set中存在与否。
  • Set.prototype.clear()：清除所有成员，没有返回值。

示例：

遍历操作

  • Set.prototype.keys()：返回键名的遍历器。
  • Set.prototype.values()：返回键值的遍历器。
  • Set.prototype.entries()：返回键值对的遍历器。
  • Set.prototype.forEach()：使用回调函数遍历每个成员。

由于Set结构没有键名，只有键值（或者说键名和键值是同一个值），所以==keys方法和values方法的行为完全一致==。

六、Map

1、简介

Map：JavaScript的对象（Object），本质上是键值对的集合（Hash 结构）。但普通的对象只能用字符串当作键；而对于Map对象，任何值（对象或者基本类型）都可以作为一个键或一个值。

关于Map的属性及方法可以直接参照Map | Mdn。

2、WeakMap和Map之间的区别？

  1. WeakMap只接受对象作为键名（null除外），不接受其他类型的值作为键名。
  2. 并且key值是弱引用，不计入垃圾回收机制。
  3. WeakMap没有遍历操作和size属性。

详细内容可参照：WeakMap与Map的区别_ruanyifeng

七、数据类型判断

  1. typeof
  2. instanceof
  3. toString
  4. constructor

说明：

  • typeof：对于基本类型，除null外，均返回正确结果；对于引用类型，除function外，均返回object。
  • instanceof：A instanceof B，判断A是否为B的实例。instanceof检测的是原型，但instanceof只能用来判断两个对象是否属于实例关系，而不能判断一个对象实例具体属于哪种类型。
  • constructor：无法用于null和undefined。函数的constructor是不稳定的，当protoype被重写后，constructor会默认为Object。
  • toString：toString是Object的原型方法。对于Object对象，直接调用toString()方法；对于其他对象，需借助call/apply。

八、类型转换

1、隐式转换

  1. 数字运算。“＋”：将表达式的值转换为String（仅当有一个是String类型）；“－”：将表达式的值转换为Number;
  2. 两次取反（!!）；
  3. if语句；
  4. ＝＝
  5. 逻辑运算符：逻辑与（&&）、逻辑或（||）、逻辑非（!）；

==隐式转换规律：==

  • ‘string + number’字符串加数字，数字会转换为字符串；
  • ‘number - string’ 数字减字符串，字符串会转换为数字，如果字符串无法转换为数字（例如”abc”、“JavaScript”），则会转换为NaN；
  • ‘string - number’字符串减数字，字符串会转换为数字，如果字符串无法转换为数字，则会转换为NaN；即’➖’优先向数字转换。
  • 乘、除运算时，也会先将字符串转换为数字。

可以巧妙地记一下，当字符串和数字同时存在时，除了加号运算符转化为字符串，其他减、乘和除都优先转化为数字。

2、强制类型转换（显式转换）

出现显式转换的情况：toString()/String()、Number()/parseInt()/parseFloat()、Boolean()等

==注意：==

  • null和undefined没有toString()方法。
  • 使用String()时，对于Number和Boolean，就是调用toString()方法；对于null和undefined，并不会调用toString()方法。
  • 使用Number()时，若字符串中有非数字，则转换为NaN。null转为0，undefined转为NaN。
  • parseInt()可以将一个字符串中的有效的整数内容去出来，然后转换为Number。如果对非String使用parseInt()或parseFloat()，它会将其转换为String然后在操作。
